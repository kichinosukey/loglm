## loglm v0.1.6

This release improves first-run guidance for coding agents launched through `loglm`.

### Highlights

- `loglm` now creates or updates the selected agent's instruction file on normal launch:
  - `AGENTS.md` for Codex
  - `CLAUDE.md` for Claude Code
  - `GEMINI.md` for Gemini
- The managed notes explain what `loglm` is, where logs are stored, and how agents should inspect past work.
- The notes mention both `loglm-decode` and `loglm-timeline`, so agents can decode raw logs or build a chronological overview when needed.

### Why This Matters

- New users do not need to manually explain `loglm`, `logs/`, or decoded logs to the coding agent.
- This is especially useful in classroom/workshop settings where students start from fresh project directories.

### Validation

- Added regression coverage for automatic `CLAUDE.md` runtime notes during Claude Code launch.

### Install / Uninstall

```bash
curl -fsSL https://raw.githubusercontent.com/ks91/loglm/main/install.sh | bash
curl -fsSL https://raw.githubusercontent.com/ks91/loglm/main/uninstall.sh | bash
```
