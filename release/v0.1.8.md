## loglm v0.1.8

This is a bugfix release focused on Claude Code log decoding and safer PII redaction.

### Fixes

- Improved Claude Code leading replay trimming.
  - Decoded logs now remove more repeated context shown after resuming a Claude Code session.
  - Claude logs prefer a matching `claude --resume <UUID>` session ID when selecting the previous log for overlap trimming.
  - This helps when unrelated Claude sessions appear between two logs from the same resumed conversation.
- Fixed PII review behavior when raw, decoded, and redacted variants are passed together.
  - `loglm-decode --review-pii ... logs/*` now avoids processing multiple inputs that target the same `*.redacted.txt`.
  - If variants conflict, `*.decoded.txt` is preferred over raw `*.txt`, and raw `*.txt` is preferred over existing `*.redacted.txt`.
  - This prevents a correctly generated redacted file from being overwritten by another variant of the same log.

### Improvements

- Added exclusion strings to grouped PII candidate lists.
  - Lines beginning with `-` are treated as exclusions for that group.
  - Example:

```text
いし
-願いし
```

  - This redacts the nickname `いし` while preserving phrases such as `お願いします`.

### Validation

- Added regression coverage for Claude Code leading replay trimming with redraw variations.
- Added regression coverage for Claude Code session-ID-aware overlap trimming.
- Added regression coverage for PII exclusion strings.
- Added regression coverage for mixed raw/decoded/redacted PII review inputs.
